PROMPT DESIGN PRINCIPLES:

  1. STRUCTURED CONTRACTS
     ✓ Role, Goal, Input, Output explicitly defined
     ✓ XML tags for machine-parseable structure
     ✓ Schema definitions for all data types

  2. PRIORITY ANNOTATIONS
     ✓ critical - Must never violate (security, correctness)
     ✓ high - Important business logic
     ✓ medium - Best practices, optimization
     ✓ low - Observability, nice-to-have

  3. ERROR HANDLING
     ✓ Exact responses for each failure scenario
     ✓ Graceful degradation strategies
     ✓ Customer-friendly messages (no technical jargon)

  4. DETERMINISTIC EXECUTION
     ✓ Code/tools handle logic, not LLM reasoning
     ✓ Reproducible results for testing
     ✓ Auditability through logging

  5. OBSERVABILITY
     ✓ Logged at each phase for debugging
     ✓ CloudWatch correlation support
     ✓ Metrics and alerting ready
